# Что это
Это консольный мессендждер на python с использованием сокетов, json формата. 
Он имеет серверную и клиентскую части, что позволяет максимально контролировать соединения, например для организации чата с друзьями, для чего я его и делаю:)

# Функционал
## Команды клиента
```python
🔐 Авторизация и регистрация
      /reg <юзернейм> <пароль> <пароль>          — зарегистрироваться на сервере
      /auth <юзернейм> <пароль>                  — аутентифицироваться на сервере

👤 Профиль и общение
      /set_nick <никнейм>                        — установить отображаемый никнейм
      /msg <никнейм> <сообщение>                 — отправить личное сообщение
      /offline_sms                               — получить сообщения, пришедшие в ваше отсутствие
      /get_users                                 — посмотреть список онлайн-пользователей

📂 Файлы
      /all_files [количество]                    — показать последние файлы
      /send_file <никнейм> <имя_файла> [текст]   — отправить файл из папки data/files
      /load_file <имя_файла>                     — скачать файл, присланный вам

🛠️ Системные
      /help                                      — показать эту справку
      /exit                                      — выход из программы
```
## Команды админа сервера
```python
      /users                      — посмотреть словарь с пользователями
      /close_server               — безопасное закрытие сервера
      /help                       — эта справка
```
# Технологии
В проекте используются:
- **socket** — для сетевого взаимодействия
- **threading** — для параллельной обработки подключений и команд
- **json** — для обмена данными между клиентом и сервером
- **hashlib + secrets** — для хеширования паролей с солью
- **dataclasses** — для удобной работы с данными
- **декораторы** — для проверки авторизации и логирования
- **os, time, datetime, logging, base64** — для сопутствующих задач
# Как пользоваться
Основными исполняемыми файлами являются `Server/main_server.py`, `Client/main_client.py` для сервера и клиента соответственно.
Для корректной работы необходимо всем участникам соединения находиться в локальной сети (можно возспользоваться различными сервисами), либо иметь белые IP-адреса
(ну или шаманить, так как есть способы прямого подключения без белого ip, но я не стал париться). Далее прописать в аргументы экземпляров главных классов Server и Client ip и port сервера,
что на клиенте, что на сервере. Например:
```python
from core.Server import Server


server = Server('127.0.0.1', 1234)
```
```python
from core.Client import Client


client = Client('127.0.0.1', 1234)
```
Для запуска сервера и клинета локально. То есть для релаьного использования надо установить свой ip и любой порт.

P.S. Как я понимаю, достаточно белого ip для сервера, так как именно на его ip ориентируются клиенты.
